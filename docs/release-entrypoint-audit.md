## 本轮发现与修复

- 问题：`任务` 在顶部栏、Hero 操作区和左侧导航中同时出现，浏览器可访问名称容易被解析成多个同名按钮。
- 修复：顶部任务按钮增加 `aria-label="查看任务中心，当前 N 个任务"`，Hero 任务按钮增加 `aria-label="打开任务中心"`，左侧导航 `任务` 保持唯一入口名。
- 复测：`getByRole('button', { name: '任务', exact: true })` 的可见匹配数从 2 降为 1，点击后可见 `任务中心`。

## 仍需后续覆盖

- 深层生成表单的真实提交链路仍需单独验收，包括视频、图片、影视创作和工作流画布导出后的 API 调用。
- BYOK 只验证了入口可见，尚未在本轮输入测试 API Base / API Key 或调用 `/api/provider/test`。
- `/research` 是研究页面，发布时建议继续确认其中平台对比内容是否为最新资料。

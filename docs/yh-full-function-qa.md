# TashanScene工作室全功能可用性回归

更新时间：2026-06-16 15:27（Asia/Shanghai）

## 本轮范围

本轮按自动化 M7 执行入口级可用性回归，目标不是消耗真实模型额度，而是确认用户从首页、导航和直达路由能进入核心功能面，并记录阻塞点。参考目标仍是 ViMAX、Toonflow-app、ArcReel 的生产链路完整性，以及吉梦、可灵、海螺、Runway 这类商业产品的入口清晰度。

## 本轮发现

1. 图片入口原本能工作，但首页行动卡缺少稳定 `aria-label/title`，自动化和无障碍验证只能通过重复文本模糊命中，容易点错到画廊卡片。
2. 左侧“素材 / 精灵”等导航已经有 `title`，可稳定进入；但不同入口的可访问名称策略仍不统一，后续应统一成显式 `aria-label`。
3. 当前没有执行真实模型生成；最长稳定生成能力本轮未测，仍停留在入口级和任务接口级验证。后续需要按低成本阶梯从文本、图片到单镜头视频逐级测。

## 后续 M7 验收口径

- 每次前端入口改动后，必须至少回归首页、视频、图片、精灵、影视、素材、任务、设置、画布、研究 10 个入口。
- 前端点击通过不代表生成链路通过；模型调用仍要按 M1/M2/M3 阶梯单独验证。
- 若入口文案继续调整，保留路径应以 `aria-label/title/data-testid` 为准，避免只依赖可见文案。
